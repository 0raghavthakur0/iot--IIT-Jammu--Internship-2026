# Q20 – setup() and loop() Functions in Arduino

## 1. What is setup()?

The `setup()` function runs only **once** when the Arduino is powered on or reset.

It is used to initialize the hardware and configure pins.

Typical tasks performed in `setup()`:

- Configure input/output pins using `pinMode()`
- Start Serial communication using `Serial.begin()`
- Initialize sensors and modules

### Example

```cpp
void setup() {
  pinMode(13, OUTPUT);
  Serial.begin(9600);
}
```

---

## 2. What is loop()?

The `loop()` function runs continuously after `setup()` finishes.

It contains the main program logic and keeps repeating as long as the Arduino has power.

### Example

```cpp
void loop() {
  digitalWrite(13, HIGH);
  delay(1000);

  digitalWrite(13, LOW);
  delay(1000);
}
```

---

## 3. What happens if a long delay() is used inside loop()?

A long `delay()` stops the Arduino from performing any other task until the delay finishes.

During this time:

- Sensor readings are delayed.
- Button presses may be missed.
- Communication becomes slow.
- The system responds slowly to changes.

---

## 4. How does it affect sensor responsiveness?

Suppose a sensor checks for fire every second.

If the program uses:

```cpp
delay(5000);
```

the Arduino waits **5 seconds** before reading the sensor again.

As a result:

- Important events may be detected too late.
- Real-time performance decreases.
- IoT systems become less reliable.

---

## 5. What is the non-blocking alternative?

The recommended alternative is the `millis()` function.

Instead of stopping the program with `delay()`, `millis()` checks how much time has passed while allowing the Arduino to continue performing other tasks.

### Example

```cpp
unsigned long previousTime = 0;

void loop() {

  if (millis() - previousTime >= 1000) {

    previousTime = millis();

    Serial.println("Sensor Read");

  }

}
```

---

# Comparison Table

| delay() | millis() |
|-----------|-----------|
| Stops the program | Does not stop the program |
| Misses sensor events | Reads sensors continuously |
| Blocks execution | Non-blocking |
| Poor for IoT | Best for IoT applications |

---

# Conclusion

The `setup()` function initializes the Arduino once, while the `loop()` function runs continuously. Long `delay()` calls reduce system responsiveness, whereas `millis()` enables efficient non-blocking programming, making it ideal for IoT and real-time embedded systems.
