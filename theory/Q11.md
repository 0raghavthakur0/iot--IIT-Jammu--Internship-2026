# Q11 – Four Layer IoT Architecture

## IoT Architecture

```
+-------------------------------------+
|        Application Layer            |
|  Smart Home, Smart City, Healthcare |
+-------------------------------------+
                ↑
+-------------------------------------+
|        Processing Layer             |
|  Cloud Server, Database, Analytics  |
+-------------------------------------+
                ↑
+-------------------------------------+
|         Network Layer               |
| Wi-Fi, Bluetooth, ZigBee, Ethernet  |
+-------------------------------------+
                ↑
+-------------------------------------+
|        Perception Layer             |
| Sensors, Actuators, Arduino, RFID   |
+-------------------------------------+
```

## Layer Explanation

### 1. Perception Layer
The Perception Layer collects data from the physical environment using sensors and actuators.

**Example:** DHT11 Temperature Sensor

---

### 2. Network Layer
The Network Layer transfers data between devices and cloud servers.

**Example:** Wi-Fi

---

### 3. Processing Layer
The Processing Layer stores, processes, and analyzes collected data.

**Example:** Cloud Database

---

### 4. Application Layer
The Application Layer provides services to users based on processed data.

**Example:** Smart Home Monitoring App

---

## Summary Table

| Layer | Function | Example |
|--------|----------|---------|
| Perception | Collect data | DHT11 Sensor |
| Network | Transfer data | Wi-Fi |
| Processing | Analyze & Store | Cloud Server |
| Application | User Services | Smart Home App |

## Conclusion

The IoT architecture allows data to move from sensors to the user through communication and processing layers, enabling smart applications.
